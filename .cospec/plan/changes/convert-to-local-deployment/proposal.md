# 变更：将项目转换为本地部署版本

## 原因
当前项目配置为云端生产环境部署，需要将其转换为本地部署版本，使开发者能够在本地环境中快速搭建和运行系统，便于开发、测试和演示。

## 变更内容
- **配置管理**：将硬编码的API密钥改为.env文件配置
- **部署架构**：配置本地生产环境（Nginx + Gunicorn）
- **数据库**：实现SQLite数据库自动初始化
- **静态资源**：将CDN链接替换为本地文件
- **依赖管理**：更新requirements.txt以支持本地部署
- **文档更新**：重写README.md，面向有经验的开发者

## 影响
- **受影响的规范**：配置管理、部署流程
- **受影响的代码**：
  - `app.py`: 修改配置类，添加环境变量支持
  - `templates/`: 替换CDN链接为本地资源
  - `requirements.txt`: 添加缺失的依赖包
  - `ReadMe.md`: 完全重写为本地部署指南
  - 新增文件：`.env.example`, `config.py`, `docker-compose.yml` (可选)
